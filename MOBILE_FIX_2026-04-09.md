# Исправление проблемы с белым экраном на мобильных устройствах

**Дата:** 2026-04-09  
**Проблемы:** 
1. Сайт qsen.ru показывает белый экран на мобильных устройствах
2. Иконки Material Icons не отображаются (показывается текст вместо иконок)

## Что было сделано

### 1. Улучшен Service Worker (public/sw.js)
- Изменена версия кэша с `qsen-v2` на `qsen-v3` для принудительного обновления
- Добавлено детальное логирование всех операций SW
- Изменена стратегия кэширования: для HTML-файлов теперь используется "Network First" вместо "Cache First"
- Добавлена обработка ошибок при кэшировании
- Улучшена обработка ошибок fetch запросов

### 2. Исправлен LanguageContext (src/contexts/LanguageContext.jsx)
- Добавлена защита от ошибок localStorage (важно для приватного режима браузера)
- Все операции с localStorage обернуты в try-catch
- Добавлен fallback на русский язык при любых ошибках
- Улучшено логирование ошибок

### 3. Улучшен index.html
- Добавлено принудительное обновление Service Worker при регистрации
- Добавлен расширенный обработчик глобальных ошибок
- Добавлен debug-режим: при добавлении `?debug` в URL ошибки будут показываться на экране

### 4. Исправлены Material Icons (public/fonts/material-icons.css)
- **КРИТИЧЕСКОЕ ИСПРАВЛЕНИЕ:** Изменен путь к шрифту с относительного `./material-symbols-outlined.woff2` на абсолютный `/fonts/material-symbols-outlined.woff2`
- Относительный путь мог неправильно резолвиться на мобильных браузерах
- Теперь шрифт загружается корректно независимо от страницы

### 5. Созданы диагностические страницы

#### /test-mobile.html
Страница для диагностики проблем на мобильных:
- Показывает информацию о браузере и устройстве
- Тестирует загрузку всех ресурсов (CSS, JS, шрифты)
- Проверяет доступность localStorage/sessionStorage
- Выводит все ошибки JavaScript в видимый лог

#### /clear-cache.html
Страница для очистки кэша:
- Удаление всех Service Workers
- Очистка всего кэша браузера
- Очистка localStorage и sessionStorage
- Удобные кнопки для каждой операции

## Как протестировать

### Шаг 1: Очистка старого кэша
На мобильном устройстве откройте:
```
https://qsen.ru/clear-cache.html
```

Нажмите кнопку "Очистить всё", затем "Перейти на главную"

### Шаг 2: Проверка основного сайта
Откройте:
```
https://qsen.ru
```

Сайт должен загрузиться корректно с иконками.

### Шаг 3: Диагностика (если проблема осталась)
Откройте:
```
https://qsen.ru/test-mobile.html
```

Эта страница покажет:
- Какие ресурсы не загружаются
- Какие ошибки JavaScript возникают
- Доступен ли localStorage

### Шаг 4: Debug-режим (для детальной диагностики)
Откройте:
```
https://qsen.ru?debug
```

В этом режиме все ошибки JavaScript будут показываться прямо на экране красными/оранжевыми полосами вверху страницы.

## Возможные причины проблемы

1. **Старый Service Worker кэширует устаревшую версию**
   - Решение: Открыть /clear-cache.html и очистить всё

2. **localStorage заблокирован (приватный режим)**
   - Решение: Теперь код работает без localStorage

3. **Ошибка в JavaScript на старых мобильных браузерах**
   - Решение: Открыть /test-mobile.html для диагностики

4. **Проблемы с загрузкой ресурсов (CORS, 404)**
   - Решение: Открыть /test-mobile.html для проверки

5. **Иконки не загружаются (относительный путь к шрифту)**
   - Решение: ИСПРАВЛЕНО - теперь используется абсолютный путь

## Технические детали

### Service Worker стратегия
- **HTML файлы**: Network First (всегда пытаемся загрузить свежую версию)
- **Статические ресурсы**: Cache First (быстрая загрузка)
- **Автоматическое обновление**: При каждой загрузке страницы

### Обработка ошибок
- Все критические операции обернуты в try-catch
- Ошибки логируются в консоль
- В debug-режиме ошибки показываются на экране

### Совместимость
- Работает без localStorage (для приватного режима)
- Работает без Service Worker (для старых браузеров)
- Graceful degradation для всех функций

### Material Icons
- Шрифт: Material Symbols Outlined (локальный)
- Размер: 3.2 MB (woff2)
- Путь: `/fonts/material-symbols-outlined.woff2` (абсолютный)
- CSS класс: `.material-symbols-outlined`

## Следующие шаги

Если проблема сохраняется после всех исправлений:

1. Проверить /test-mobile.html на мобильном устройстве
2. Сделать скриншот страницы с ошибками
3. Проверить консоль браузера (если доступна)
4. Попробовать в другом браузере на том же устройстве
